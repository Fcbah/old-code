Developing Applications on MacOS X

CocoaSharp supports for MacOS X 10.3.4 and later. CocoaSharp is currently in very early developement as the release number 0.1 implies. However, a lot of functionality is already present and it is quite usable in its current form.

Installing CocoaSharp

Visit the download page to find the latest MacOS X package. The CocoaSharp framework does require that you have the Mono framework installed. You can download the Mono framework from the same page.

This package installs as a framework to /Library/Framework (the same way the Java packages are installed). Symlinks are created for the executables in /usr/bin.

Developing applications on MacOS X

CocoaSharp is aimed at .Net/Mono developers that want to allow their users to have a native Mac OS X application experience. CocoaSharp allows developers to call Cocoa APIs in native C# code using dot notation. Developing Cocoa# applications will be just like developing Mono application except that you are allowed to call Cocoa and Foundation APIs.

Once your application has compiled (and the bugs are fixed) it will function like any other Cocoa application. The one exception to this is that Cocoa applications must run on thread 0. This is currently not possible for Mono applications. As a temporary workaround a CocoaSharpLoader has been included in the Commands directory inside the Framework. The CocoaSharpLoader is in charge of the actual launching of your application so that it runs on thread 0.
