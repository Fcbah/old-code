XMLとは?

XML (eXtensible Markup Language)はW3C (the World Wide Web Consortium)によって策定されたウェブ標準です。.

データファイルや、文書ファイルやメッセージ送信などのいくつかのフォーマットで広く使われています。例えば、XHTMLがそうですし、Officeファイルフォーマット（全てのOpenOffice.orgのファイルはXmlです）もそうです。たくさんのツールはAPIがxmlの処理に利用できます。XMLは簡単に手作業で編集できます。XMLは半構造的データフォーマットで、その柔軟性のために、他のシステムへの変換ができます。

.NETでのXMLプログラミング

ほとんど.NETフレームワークに由来する、いくつものXML操作APIがあり、それらは：

  • XMLリーダーを使ってXMLドキュメントを読み込み、検証する
  • XMLライターを使ってXMLドキュメントを書き出す
  • Document Object Modelを使ってXMLドキュメントを操作する
  • XMLスキーマを使ってXMLドキュメント構造を作る
  • ストラクチャードXMLドキュメントを、XMLシリアライザーでCLRタイプに変換する
  • 強力なクエリ言語のXPATHとXPathNavigatorで、XMLドキュメントを全検索する
  • XSLTスタイルシートとXslTransformで、XMLドキュメントを他のXMLドキュメントに変換する
  • ADO.NETでXMLドキュメントを一組のデータテーブルとして扱う

われわれは、Microsoftの.NETフレームワークがSystem.XML.dllで実装しているほとんど全てをサポートしています。また、多くのADO.NETクラスも同様にサポートします。

一般テクノロジー

XMLはいくつもの「ノード」から成るツリー構造として理解できます。 DOM構造モデルが、「ノード」をほとんど同じ概念に基づいて説明しているので、こちらを確認してください。

"XML Information Set"や"Information Set"または、省略された"infoset"という言葉を、ノードの情報を表すために用います。これは基本的に W3C XML Information Set Recommendationと同じです。
