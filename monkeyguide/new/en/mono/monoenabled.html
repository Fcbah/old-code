Platforms supported

Mono is available on Windows, Linux, MacOS X, Solaris and FreeBSD.

The Mono compilation engine is easy to port to new platforms and provides a solid framework for code optimization. It also includes a code pre-compiler to reduce startup time. For some platforms, a faster interpreter called Jitter is available as well. See the Mono web page for the latest platform support listings.

Mono includes two execution systems: one is based on an interpreter (the mint command) and another one is based on an advanced code geneerator (the mono) command).

The mono command can be used in Just-in-Time (JIT) mode by typing mono followed by the program to run, or it can be used in batch mode, by passing the --aot flag (AOT stands for `Ahead Of Time'). In JIT-mode not all optimizations are turned on, since some of them can be quite time consuming. For this reason, when using it in batch mode, it is recommended that you use the -O=all switch to improve the code generation, for example: 


    $ mono --aot -O=all library.dll

The above will leave the native version of the code in `library.dll.so'

Toolkit Support

Mono currently has support for GNOME and GTK toolkits through the GTK# package, and for Windows native toolkits through the System.Windows.Forms APIs.
