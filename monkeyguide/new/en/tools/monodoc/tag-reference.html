This document explains how to document Mono classes using the ECMA-style XML format. This format is used for the Mono class libraries as well as GTK#.

See the directory monodoc/class in the subversion repository for examples. I'm just going to list each tag's usage here one by one. This is based on the ECMA DTD.

Changes from the original specifications:

If you generated your ECMA-style docs before July 4, 2003, then you need to make sure your docs follow these new rules: Ref and out parameters are distinguished. See the new Parameter/@RefType attribute. Delegate parameters and return values are added with the tags Parameters and ReturnValue within the Type element. Event types are added in the ReturnValue elements of event members. For properties, the Parameters list should not include the implicit "value" parameter. Email tauberer@for.net if you need a lot of docs updated.

When I preceed a name with an @ sign, I'm talking about an attribute of a tag (as in XPath).

Tags for "free-form" documentation are listed at the end.

The Monodoc browser does not currently honor all Excluded tags

Type - The top-level element for each XML file.

@Name - The name of this type (eg "String").

@FullName - The namespace + name of this type (eg "System.String").

@FullNameSP - ?

TypeSignature - A language-specific signature for the type. Every type should have at least a C# signature. Specify the language in @Language and put the signature in @Value. (eg "public static class String : System.Object;")

MemberOfLibrary - "BCL" for classes in corlib.

AssemblyInfo - Describes the assembly that contains this type.

AssemblyName - The name of the assembly (eg "mscorlib").

AssemblyPublicKey - The public key (eg "00 00 00 00 00 00 00 00 04 00 00 00 00 00 00 00")

AssemblyVersion - The version (eg "1.0.x.x").

AssemblyCulture - The culture (eg "none").

Attributes - The attributes on the assembly. See the Attributes section below.

Base - Describes this type's base type.

BaseTypeName - The full class name of the base type. (eg "System.ValueType")

ExcludeBaseTypeName - ?

ExcludeLibraryName - ?

Interfaces - A list of the interfaces implemented by this type.

Interface - An implemented interface.

InterfaceName - The full class name of the interface. (eg "System.ICloneable")

Excluded - 0 if this interface isn't to be excluded from anything.

Attributes - A list of the custom attributes on this type. See below.

ThreadingSafetyStatement - Statement about thread safety. Optional. Free-form documentation.

Members - A list of zero or more Member tags.

TypeExcluded - 0 if this type is not to be excluded from anything.

Docs - Explained at the end.

Parameters - If this type is a delegate, the delegate's parameters. (See the Parameters explanation below.)

ReturnValue - If this type is a delegate, the delegate's return value. (See the ReturnValue explanation below.)

Attributes - A list of the custom attributes on a type or member. Contains zero or more Attribute tags.

Attribute - An attribute on this type.

AttributeName - A string that shows how the attribute was constructed for the type. (eg "CLSCompliantAttribute(true)")

Member - Represents a member of a type.

@MemberName - The name of the member (eg "ToString")

MemberSignature - Like TypeSignature, but for members. (eg "public static byte Parse(string s, NumberStyles style, IFormatProvider provider);")

MemberType - The type of the member. One of: Constructor, Method, Property, Field, Event.

Attributes - See above.

ReturnValue - Describes return value or value type of this member. Applicable for methods, properties, fields, and events. For events, this is the type of the event.

ReturnType - The full class name of the return value or value type. Generated files may not haveReturnValue tags for events. This tag needs to be added for events so that Monodoc can display event types hyperlinked.

Parameters - A list of zero or more parameters for this member. Applicable for constructors, methods, and properties with indexers. For properties, do not include the implict value parameter of the set accessor. This is expressed in the ReturnValue section. Use the parameters that go with the get accessor.

Parameter - A parameter.

@Name - The name of the parameter.

@Type - The full type name of the parameter. By-reference parameters (ref or out) should end with an ampersand. (eg "System.String]&" for "ref string[")

@RefType - Either "ref" or "out", if this parameter is by-reference. This is not a part of the ECMA spec and so may not be in template files. Monodoc will list out parameters as ref if RefType is not set to "out".

Attributes - The attributes on this parameter. See the Attributes section above. Optional.

MemberValue - The value of this member, if it is constant. Optional. (eg "255")

Excluded - 0 if this member is not to be excluded from anything.

ExcludedLibrary - ?

Docs - Explained at the end.

Docs - This node contains a variety of sections of free-form documentation. The purpose of each section is described here. Each section can contain free-form documentation tags, which are documented after. All sections are optional, and some can be repeated more than once (as indicated).

summary - Summary description of the type/member.

altmember - Put a full class name in its cref attribute (T:Full.Type.Name) to provide a see-also link.

altcompliant - ?

param - Describes a parameter to the method/property. Repeat for each parameter. Put the name of the parameter in the name attribute.

returns - Describes the return value of the method.

value - Describes the value of the property or field.

exception - Describes an exception that can be thrown by the method or constructor. Repeat for each exception. Put the full class name of the exception in its cref attribute (eg "T:System.Exception"), and put the conditions in which the exception is thrown in the body of the tag.

threadsafe - Currently ignored by Monodoc.

remarks - General remarks about the class.

example - An example usage of the type or member. (No preformatting is applied here.)

permission - Describes the permissions needed to access the type or member. Repeat for each permission. Put the full class name of the permission in its cref attribute, and put a description in the body of the tag.

Free-Form Documentation The following tags can be used in free-form documentation tags. The description of each tag specifies what the tag transforms into. The body of these tags can also contain free-form documentation tags.

para - A paragraph.

paramref - A reference to a parameter of the constructor/method/property. Put the name of the parameter in @name.

block - A subsection of documentation. Specify the type of the subsection in @type: "note" for notes, "behaviors" for quirks of the member, "overrides" for notes to inheritors, and "usage" for usage notes.

c - Monospaced font for code excerpts. Use para tags to split lines if necessary.

code - A block of code. Specify the language in @lang. Newlines are preserved.

list, when @type = "table" - Tabular data. Here's an example:

        <list type="table">

                <listheader>
                        <term>First Col Header</term>
                        <description>Second Col Header</description>
                        <description>Third Col Header</description>

                </listheader>
                <item>
                        <term>First Row First Col</term>
                        <description>First Row Second Col</description>
                        <description>First Row Third Col</description>

                </item>
                <item>
                        <term>Second Row First Col</term>
                        <description>Second Row Second Col</description>
                        <description>Second Row Third Col</description>

                </item>
        </list>

list, when @type = "bullet" or "number" - A bulletted or numbered list. Here's an example.

        <list type="bullet">
                <item><term>First item.</term></item>

                <item><term>Second item.</term></item>
        </list>
see - A link to another page in the documentation. Use the cref attribute to link to a type or member or the langword attribute to specify a language keyword. The body of the tag is ignored. Cref attributes have the form: a one-letter prefix (N, T, C, M, P, F, E), a colon, and a value. Here are some example crefs

"N:" for Namespaces, example N:System

"T:" for Types (classes, structs, interfaces, enumerations), example: T:System.Byte

"C:" for Constructors, example: C:Gtk.Button()

"M:" for Methods, example: M:System.String.Substring(System.Int32,System.Int32) (the argument list is optional)

"P:" for Properties, example: P:System.AppDomain.CurrentDomain

"F:" for Fields, example: F:Gtk.TreeIter.Zero

"E:" for Events, example: E:Gtk.Button.Clicked

Common langword usages: <see langword="null"/>, <see langword="true"/>.

subscript - Subscript.

superscript - Superscript.

onequarter - Transforms to a 1/4 symbol.

pi - Transforms to a pi symbol.

theta - Transforms to a theta symbol.

Other tags will be left unchanged.
