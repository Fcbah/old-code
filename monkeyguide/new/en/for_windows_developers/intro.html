If you're reading this chapter, we'll assume that you already know something about .NET. Mono allows you to work with existing .NET concepts, but it introduces a new set of libraries for working with Linux GUIs. These are familiar to Linux GUI developers who've worked with the GNOME tool stack, but you'll need to learn them. Here's a list:

  • art : a 2d graphics library.
  • atk : accessibility toolkit.
  • gconf : maintains per
  • user configuration files, much like the Windows Registry.
  • gda : GNOME data access
  • gdk : a library that sits between the window system (X11) and Gtk.
  • glade : Glade is a UI-building tool.
  • glib : Interface to the X11 system.
  • gnome : desktop environment libraries on top of GTK. Includes concepts like icons, fonts, printing, Druids (wizards)

  • gnomeprint : not yet implemented

  • gtkhtml : an HTML renderer.
  • panelapplet : for writing toolbar/panel widgets.
  • pango : a globalized text renderer.
  • parser : tools.
  • rsvg : bindings to librsvg, a scalable vector graphics library.
