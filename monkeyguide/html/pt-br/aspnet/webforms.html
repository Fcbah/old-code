The "Hello World" example is coded in old ASP / PHP style. It is supported, but results in "spagetti-code". That means the code is mixed with the design, leading to difficult reading and problems when the design should be changed.

ASP.NET offers a new programming model: WebForms. ASP.NET separates code from design. How is this done? If you have ever written desktop applications for GTK+, Qt or Windows you'll be used to widgets like buttons are accessible as objects. ASP.NET finally gives you this comfort also for WebApplications. Why would you like to write anymore for the client if not really needed?

<%@ Page Language="C#" %>

<% // This is executed at page startup

public void Page_Load()
{
  label1.InnerHtml = now(); 
}

%>

<html>
<body>
<h2>Hello Mono!</h2>
<p><span id="label1" runat=server/></p>
</body>
</html>
The difference to helloworld! can be seen. The code - the date "now()" method is removed from the html. Just like with GTK+ a label is used to put the HTML in.
