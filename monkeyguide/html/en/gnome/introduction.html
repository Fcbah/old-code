GNOME Development with Mono

What is Gtk#?

This chapter covers use of the GNOME development platform with C# and .NET. The breadth of features and stability of the GNOME development platform and the rapid development tools from .NET make a powerful combination, as you will see in the following exercises. Among the tools available to you as a developer are the Glade user interface design tools, the GnomeDB database access system, the Gecko HTML engine from Mozilla, and the GConf settings registry. That means that, for example, you can write a web browser in just a few lines of code, as we do in the second Glade.NET exercise.

As we explore the use of the GNOME tools, be aware that you can use one library without committing yourself to the rest of them, or committing yourself to the GNOME desktop environment or even any particular OS. While Gtk# is primarily designed for development of Linux and UNIX applications, many of the Gtk# libraries are fully functional on Windows and Macintosh OS X as well. Where possible, examples will be cross-platform functional, and we will note libraries and function calls that do not translate across platforms.

Installing the Gtk# Development Environment

The standard GNOME packages you will need are probably already part of your distribution. If not, you can install them from Red Carpet and OpenCarpet servers, package repositories like RPMFind or Debian.org, or with the one-step GARNOME build script.

A comfortable base set of packages for a Linux developer would be:

  • glib and glib-devel, for basic convenience functions
  • pango and pango-devel, for text-rendering with full UTF-8 compliance
  • atk and atk-devel, for accessibility
  • gtk2, gtk2-devel, gtk-sharp and gtk-sharp-devel, for the actual GUI toolkit
  • ORBit and ORBit-devel, for CORBA object communications
  • gtkmozembed (sometimes packaged as part of mozilla-devel), for Mozilla embedding
  • gtksourceview and gtksourceview-devel

If you are using Monodevelop, you have most of these libraries already.

Windows developers will also need the Cygwin and mingw-cc tools.

Installing the Glade Interface Builder

The Glade UI tool is not required but is strongly recommended. It is included with most Linux distributions, and a Windows version is also available.

The Glade application allows you to use simple drag-and-drop tools to lay out your user interface, and then saves the UI as an XML file. Then, with a single call to the libglade library, the user interface is generated and attached to the data structures and functions you have created in your code. Because user interface generation happens at runtime, you can even adjust the UI without recompiling.
