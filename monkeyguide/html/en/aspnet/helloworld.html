How does it work?

Like in ASP or PHP, the web server does not send immediately the content of the file first.aspx to the browser, but processes any code , written in <% %> tags. The browser receives pure HTML. 
<%@ Page Language="C#" %>

<html>
<body>
<h2>Hello Mono!</h2>
<p><%Response.Write(now())%></p>
</body>
</html>
This example will result in this source sent to the browser: 
<html>
<body>
<h2>Hello Mono!</h2>
<p>21.12.2002</p>
</body>
</html>
The code in the %-%-tags can be any mono supported language with CodeDom implementation, currently only C#. Also, you can make use of the complete class library, that should fit any needs.

To insert output in the page, you must use Response.Write(), instead of Console.Write().
