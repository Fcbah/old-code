Ensuring Cross-Platform Compatibility

If you build your application with cross-platform execution in mind, it will be more likely to succeed. Even if your development tools are neutral, you need to make sure that you are aware of the differences and similarities among the various target platforms.

Here are several rules you can follow to build cross-platform applications:

  • Use relative paths when naming files. The file C:\Program Files\yourapplication does not exist on Linux, and /usr/lib/ isn't available under Windows.
  • In the same vein, try to avoid platform-specific resources like the Windows registry (Microsft.Win32.Registry).
  • Given the choice between Gtk# and System.Windows.Forms, use Gtk#. Gtk# for Windows is better than System.Windows.Forms for Linux.
  • Only use Platform Invoke (P/Invoke) if you really need it. If possible, use a library that is available on UNIX and Windows.
  • If you do use P/Invoke or another platform-specific resource, don't just assume it exists. Instead, check for it and provide a clear error message if it fails.
