Break

Break is a statement used for stopping a loop. Just as its name, it breaks a loop in a certain point adn then continues to the next statement. It has the form: 

 loop {
	statements
	break;
	statements not executed
 }

A small example of its use is the next program: 

//
// BreakExample.cs
//
// Description:
//      A first example of the use of break
//

using System;

 public class BreakExample {

         public static void Main ()
         {

                 //
                 // Break in a for loop
                 //
		 for (int i = 1; i < 10; i++) {
                         // Statements
                         Console.WriteLine (i);

			 break;
                         
                         //Statements not executed
                         Console.WriteLine ("This message should not appear !");
		 }

         }
         
 }



1

It only prints the number 1. This behavior of the loop is caused by the break statement in the middle of the body of the for loop. Thus the next instruction is not executed never.

However, most of the times it is used when you want to add a special condition in your loop; for example, when an anormal situation appears and then it is necccessary to stop the loop. The next example will make clear this situation: squares of the integer from 1 to 20 must be printed, and in a second loop, the squares of the integer from 11 to 20. 

//
// BreakExample2.cs
//
// Description:
//      An example of the use of the break statement
//

using System;

 public class BreakExample {

         public static void Main (string [] args)
         {

                 int i;

                 //
                 // Break with FOR
                 //
                 for (i = 1; i <= 20; i++) {
                         if (i > 10)
                                 break;
                         Console.WriteLine ("{0} = {1}", i, i*i);
                 }

                 Console.WriteLine ();

                 //
                 // Break with WHILE
                 //
                 i = 11;
                 while (i <= 20) {
                         if (i > 15)
                                 break;
                         Console.WriteLine ("{0} = {1}", i, i*i);
                         i++;
                 }

                 Console.WriteLine ();

         }

 }

What happened? The break statement in the for loop stopped the normal execution when found a value in i bigger than 10. And in the while loop, a similar situation appears: the loop was stopped when a value bigger than 15 was found.

A last example will clarify the things even more. Let's imagine that a user shell must be implemented with C#. So we are going to read every thing the user types (using a loop). And only if the user types the exit keyword (keyword in our shell, not in C#), the program will finish its execution. Then, a break statement will stop the loop when this condition becames true. 

//
// Break3.cs
//
// Description:
//      A third example of the use of break
//

using System;

 public class BreakExample {

         public static void Main ()
         {

                 string input;

                 do {
                         Console.Write ("[Mono Shell]$ ");
                         input = Console.ReadLine ();

			 //
			 // Erase the spaces at the end/begin
			 // and convert to lower case
			 //
			 input = input.Trim ().ToLower ();

                         //
                         // If the user typed 'exit', 
                         // then finish the do/while loop
                         //
                         if (input == "exit")
                                 break;
                         
                 } while (true);

         }
         
 }


[Mono Shell]$  hola
[Mono Shell]$  e X it
[Mono Shell]$ EXIt^[[D^[[D^[[D
[Mono Shell]$     ExiT

The do/while loop will continue working if the user never types exit. That condition, input == "exit" could have been in the condition of the loop (and not only true), but this is en example that showed that, since we wanted to work with the user input directly, it was more convenient to check for the conditions inside.
